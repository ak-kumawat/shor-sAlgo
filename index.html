How It Works

Shor's Algorithm reduces integer factorization to period-finding — a task quantum computers can solve exponentially faster using superposition and the Quantum Fourier Transform. It is a hybrid classical–quantum procedure.

  • 1
    Choose random a  ClassicalSelect a random integer a, where 1 < a < n, coprime to n.
  • 2
    GCD check  ClassicalCompute gcd(a, n). If > 1, you found a factor immediately — no quantum needed.
  • 3
    Quantum period finding  QuantumQuantum circuit evaluates f(x) = aˣ mod n for ALL x simultaneously using superposition, then applies QFT to extract the period r.
  • 4
    Validate period  ClassicalIf r is odd or a^(r/2) ≡ −1 (mod n), restart with new a. Success probability > 50%.
  • 5
    Compute GCD  ClassicalCalculate gcd(a^(r/2) + 1, n) and gcd(a^(r/2) − 1, n) using Euclidean algorithm.
  • 6
    Extract factors  ClassicalNon-trivial results are the prime factors p and q. Verify p × q = n.
Mathematical Formulation

Modular exponentiation function

f(x) = aˣ mod n

Core function whose period we seek. Evaluated for all x simultaneously in superposition.

Period definition

aʳ ≡ 1 (mod n)

r is the smallest positive integer satisfying this — the cycle length of f(x).

Factor extraction via GCD

gcd(a^(r/2) ± 1, n)

Once r is found (even, non-trivial), these GCD calculations yield the prime factors.

Conditions for success

r even & a^(r/2) ≢ −1 (mod n)

If conditions fail, pick a new random a and retry. Expected attempts: 2–3.

Quantum Circuit

Three stages run on the quantum computer; initialization and post-processing run classically.

QFT — Interference Visualization

The Quantum Fourier Transform extracts the hidden period r using interference. Correct-period states interfere constructively (tall peaks); all others cancel out (destructive interference).

Peaks appear at multiples of N/r — revealing the period. Green = period multiple; grey = cancelled by destructive interference.

Quantum Superposition

Classical bits are always 0 or 1. Qubits can be both simultaneously — allowing a quantum computer to evaluate f(x) for all 2ⁿ values of x at once.

Classical vs Quantum

The Maze Analogy

Classical: Walk one path, hit a wall, backtrack, try another. Tries each x one by one — O(√N) attempts.

Quantum: All paths explored simultaneously. One "ghost" per path — the correct one is amplified by QFT, wrong ones cancel out. Period found in polynomial time.

What is at Risk?

Shor's Algorithm breaks any cryptosystem whose security relies on the hardness of integer factorization or discrete logarithms.

System Used for Status
RSA HTTPS, email, banking Broken by Shor's
ECC / ECDSA TLS, code signing, Bitcoin Broken by Shor's
Diffie-Hellman Key exchange (TLS) Broken by Shor's
AES-128 Symmetric encryption Weakened (Grover's — double key size)
AES-256 Symmetric encryption Safe (128-bit quantum security)

Harvest Now, Decrypt Later: Adversaries may be capturing encrypted traffic today to decrypt once a cryptographically relevant quantum computer exists. Any data requiring long-term secrecy is already at risk.

Current Hardware Limitations

Shor's Algorithm is theoretically complete. The hardware is not. Breaking RSA-2048 requires ~20 million stable, fault-tolerant qubits. We have ~1,000 noisy ones.

Post-Quantum Cryptography (PQC)

NIST finalized four post-quantum standards in 2024. These algorithms are secure against both classical and quantum attacks because they rely on mathematical problems with no known quantum speedup.

NIST Standard · Key encapsulation

CRYSTALS-Kyber (ML-KEM)

Lattice-based. Replaces RSA/ECDH for key exchange. Based on Module Learning With Errors.

NIST Standard · Signature

CRYSTALS-Dilithium (ML-DSA)

Lattice-based digital signatures. Replaces RSA-PSS and ECDSA.

NIST Standard · Signature

FALCON (FN-DSA)

Fast-Fourier lattice signatures. Compact size, suited for constrained devices.

NIST Standard · Signature

SPHINCS+ (SLH-DSA)

Hash-based signatures. Most conservative: security based only on hash functions.
